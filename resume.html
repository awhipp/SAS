    • Meesh Quest (A MapQuest Derivative)

      • August 2013 to Decemeber 2013

      To encompass my data structures course I was given a semester-long, three-part project:

      First Task: Implement a "map quest" program with three parts. First, a data dictionary which was composed using java.util.treemap. Second, I created a spatial map by implementing our own derivation of a polygonal region (PR) quadtree. The final part was an adjacency list of the data objects to determine weights. The program had an input of an XML document, and was parsed accordingly, calling commands and creating the map. The three parts were used in conjunction together.

      Second Task: Using the code you already had constructed I reworked each structure to implement new data structures I had learned in class. In this sense I replaced the treemap with our own AVL tree, this AVL tree had a variable passed in which determined how many levels were needed to be offbalance before a rotation would occur (the standard AVL tree has the balance factor of 1). Next, I changed the PR Quadtree to a polygonal map (PM) quadtree. Finally the adjacency list included a derivation of Dijkstra's to determine shortest path.

      Final Task: The implementations for PM1 and PM3 were done (each has a different level of restrictions on what is considered valid). Originally it was just a PM3. Now, to get a sense of what went behind the framework for the TreeMap class, we had to have our AVL tree implement the SortedMap interface which enabled us to compare between a treemap and our AVL tree. This allowed us to call such functions such as avl.entryset(), avl.keyset(), avl.values() and compare them to the corresponding treemap methods.
